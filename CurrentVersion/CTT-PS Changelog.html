v51.0

Performance Improvements:

• This script has had an enormous bottleneck since the inception of the GUI. The texture list was created using a "ListView" item, which is the grid that shows the texture name, dimensions, aspect, status, and so on. What I failed to realize is that a ListView is EXTREMELY slow to update and add entries to. So I tried to implement a "DataGridView" instead, and the performance is lightyears beyond what it was. Expect massive speed improvements overall to every single operation that updated the texture list.
• Counting textures when pressing the [Start] button is now done asynchronously from the script's main thread. This allows the "Master Loop" to start before the count is finished, so the GUI does not freeze up for a few seconds before the loop even starts. Before the count is finished, the total textures and progress percentage will display as (??). When the count is finished, both are updated accordingly during the loop. This can save the user a lot of time over the long run.
• A huge oversight was fixed that could lead to an enormous bottleneck when working with thousands of textures. The problem was with the way external mipmaps were checked for and counted. If many textures (thousands) were in the same folder, the script would frequently hang-up every loop iteration. A new method has been created that is infinitely faster (literally).

General Changes:

• Add a new CTT-PS Mode to the list called Database Mode. This is similar to "Dolphin Mode" but for textures that do not have the "tex1" prefix followed by their original dimensions. The purpose of this mode is to create and utilize databases that store texture names and dimensions. It requires building a database file using the original dumps (more on that below). When running any operation in this mode, textures that are found in the database will have their original dimensions referenced.
• Add a new operation called Create Texture Size Database that will scan textures and store their name and dimensions into a database file using the extension ".tdb". This operation should only be ran on dumped textures that contain their original dimensions. When working with HD versions of these textures while in Database Mode, the current database will be referenced to find the original dimensions of these textures, which allows calculating things such as the "scale" and "aspect". It will also allow running these textures through operations and options that usually only work with "Dolphin" textures.
• Reimplement Venomalia's Dolphin TextureExtraction Tool. The operation is now called Extract Textures With TextureExtraction Tool. Auto-scrolling the texture list is disabled during this operation because of how the two programs interact with each other. I have recently discovered that scrolling the list is extremely slow, so CTT-PS can fall behind the extraction tool. Additional functionality has also been added to CTT-PS to allow further configuration.
• Support the "combined RGBA" format that TextureExtraction Tool can create. The texture output from the tool will use the prefix "RGBA_" instead of "tex1_". When CTT-PS encounters these textures, it will automatically split them into the two textures that make it up and process them with the selected operation. This currently requires the script to be linked to Venomalia's tool, but I plan on implementing a fallback path before v51.0 leaves beta.
• Modify how Scan Dolphin Textures For Issues reports issues. There has been a long-standing bug where only a single issue was reported even if the texture had multiple issues. The new system will report the number of issues, correctly display the issues in the message column, and report if the texture was copied or fixed (if enabled).
• Added an option to Scan Dolphin Textures For Issues to allow defining user acceptable scaling values. When searching textures for issues, textures that have scaling values outside of the ones the user listed will be flagged as an "issue".
• The texture list now has a new option Store Texture List Adjustments. This option saves any changes to the texture list dimensions and/or column sizes and reloads them whenever restarting the script. To restore the original texture list sizes, simply uncheck the option and restart the script or use the "Restore Defaults" button in the Options menu.
• The dialog menu shown when selecting folders such as the Input/Output folders has been replaced with its much more modern variant that is similar to the file selection dialog menu. Special thanks to Jill for reminding me of this issue.
• Add a button to Split Combined Multi-Texture to clear a CTT file loaded from memory. Before v49.0, loading a CTT file would keep the texture names loaded into memory even after loading a new texture afterwards. This behavior has been restored, so to clear the names from memory the button has been added.
• Remove the Debug option Disable counting textures when launching operations as it is no longer needed with the addition of asynchronous counting. This option only existed to serve as a bypass to the long time it can take counting textures for very large packs, so it no longer serves any purpose.
• Checking textures for transparency in any pixels (not the same as checking for alpha channel) is now done solely with ImageMagick which is much faster than the previous method. There are very few cases where this check is used, but in the rare cases it pops up it will no longer freeze up the GUI and it will complete much faster than before.
• Reduce the number of situations where PNG images are resized before being sent to Compressonator which saves time. Because Compressonator does not have the ability to resize dimensions, the image is first sent to ImageMagick to be resized. This was true even if the dimensions were not a multiple of four, and Compressonator will automatically scale dimensions up 1-3 pixels to the nearest multiple of four. For example, 63x63 are not valid DDS dimensions but Compressonator will still create a 64x64 image. So instead of resizing these images with ImageMagick before sending to Compressonator they are sent "as is" which can prevent unnecessary resizes.
• Alert the user that a path can not contain the tilde character when attempting to set the Input Path with this character.

• Implement a workaround when modifying the "Rescale Factor". It will now increment from "0.01" to "1.00" instead of "1.01" when increasing it via the numeric up/down. Since the factor can't be "0", the value of "0.01" is as low as it can get. This is not a value that should be used, but since this is the lowest possible value, its easy to obtain by scrolling/clicking the numeric up/down, which would then require the user to manually edit out the "0.01" to a desired value.
• Add a PNG option to enable preserving color data in fully transparent pixels. This used to be the default behavior since v47.0, but it was discovered by GhostlyDark that it can cause weird artifacts in some textures. The corruption was most noticeable on some single color (white) text textures, where some pixels were not getting 100% white for some reason.
• Add partial support for DDS formats A1R5G5B5 and A4R4G4D4. The script can not create them (at least not yet), but they can be converted to other formats such as PNG. I do not know if I will add support in creating them, maybe if there is some demand.

Hot fixes to Ishiiruka textures are now applied to the header after generation, which allows other image tools to open these textures. Note that BC7 material maps can not be recreated or modified.

Icons can contain several images at different sizes, so a new option has been added to specify sizes created. The maximum size an icon can be created is 256x256, anything larger is ignored.

• Add an option to Convert/Rescale/Resize operations to toggle the sharpening filter that is applied to textures when they are resized any reason. This affects all textures that are resized, including generated mipmaps and icons. It used to be always enabled by default, with no option to disable it, but decided it should be an "opt-in" option.

Functional Changes:

• The script now saves options to an external ini file [StoredOptions.ini] by default rather than saving options directly to the script. The old behavior of saving directly the script is still possible by unchecking the option [Store/load saved changes in "CTT-PS Data" folder] found under the Preferences tab of the Options menu.
• Custom Texture Tool PS now uses a global [Data] folder that can be selected by the user on first boot, or within the Option menu on the [Tool Paths] screen. This data folder is used to hold the temporary folder, the "StoredOptions.ini" file, a resource pack database, and it can be a place to put the supported programs.
• While not new to this version, adding the [Force Create Mipmaps] flag (_mm#) to the end of textures forces generating mipmaps for that texture, the # defines the number of mipmaps created. It is now possible to leave the # portion blank and just use (_mm) to force generating the maximum number of mipmaps for that texture.
• Show the [Force Create Mipmaps] option when in Dolphin Mode. It appears that textures that are not mipmap textures dumped by Dolphin ("m" in the filename) work with mipmaps. Not suggested to use this option with entire packs as it is a waste of hard drive space to create mipmaps for textures that don't need them (such as UI textures).

This used to be checked during the loop when validating textures (Special Cases), but has now been removed. If any of the variations of this error is triggered, an error message will pop up explaining which situation caused it.
• Remove option [Skip Texture if Output Exists]. A new option has been added to the [Preferences] menu called [Overwrite textures in the output path if they already exist] which does the same thing for all textures.
• Change the way the "last path" is remembered when selecting images or programs from the Get-File window. Rather than use a global path for all selections, each has its own unique way of attempting to retrieve this value so multiple "last paths" can exist. If there is no last path, in each case it will default to the scipt location.
• The "Process Selected" option has been completely rewritten to be much faster and reliable. Generating image previews is now nearly as fast as not generating them at all, and this rework also allows images from different folders that have the same name to be processed together: before each texture had to have a unique name. This rework was possible due to the changes to the loop, validation, and texture info mentioned below.

Operation Changes:

• The "Master Loop" (the primary loop each operation runs through), texture validation, and getting texture info (width, height, etc.) are all systems that were heavily intertwined and have been rewritten to be much more efficient and able to operate independent from each other. While this should not have any noticeable changes other than a slight speed increase (and may have created some new bugs), many workarounds were able to be removed.
• Add new operation [Create Materials With Material Map Generator]. This uses the python script by JoeyBallentine to generate the materials used by Ishiiruka Dolphin. Requires Python v3, other dependencies can be checked for and downloaded automatically if missing by pressing the "Check Dependencies" button found in the operation's options.
• Add new operation [Run Custom CLI Executable With CTT-PS] that allows importing any executable that supports command line input. Imported programs can be customized to accept certain file extensions, a list of commands can be created to send to the program, and the program will process files through the CTT-PS Master Loop.
• Rework how the divisor is calculated when splitting images into segments to upscale via ESRGAN/SFTGAN. The new method improves the accuracy for seamless textures that do not have dimensions that are a power of two, but do have dimensions that are even. Images that have odd dimensions (like 155x155), may still have some inconsistencies.
• New operation name: [Optimize PNG Textures with Optimizer Program]. Two more programs are now supported. The list of supported programs is now: OptiPNG, OxiPNG, Pingo, and Efficient Compression Tool. Several more options have been added to the GUI; some options are only available depending on which program is currently selected.
• The top-left selection window used in the auto-arrange feature for Mario64/Zelda64 tile textures found in the [Combine Textures] operation will now default to the folder that the textures were dropped from.
• Add a new operation called [Seperate/Combine Texture RGBA Channels]. This allows splitting a texture into images that contain only the individual channel data, and it allows combining images using individual channel data.
• Metroid Prime: A new mode! This contains operations specific to Metroid Prime 2 (for now at least).
• Metroid Prime: Add operation to Metroid Prime Mode that allows cloning base textures using the sorted fonts.
• Phyre Mode: Add python scripts by CeruleanSky to extract PKG/PKA files in Trails of Cold Steel III and IV.
• Phyre Mode: Add operation to extract PKA files, and updated PKG extract operation to support CS3/CS4.
• Phyre Mode: In the operation [Create Phyre Textures] allow force converting DXT1 textures with alpha to DXT5.

This really only affects textures with odd scaling values (non-int-scale/non-mult-4), meaning, textures with [bad] dimensions.

Before: 768x768, 384x384, 192x192, 96x96, 48x48, 24x24, 12x12, 8x8, 4x4, 1x1, 1x1
After : 768x768, 384x384, 192x192, 96x96, 48x48, 24x24, 12x12, 8x8, 4x4, 2x2, 1x1

This also improves calculations for textures with [bad] dimensions. For example, a 578x578 [m] texture...
Before: 580x580, 292x292, 148x148, 72x72, 36x36, 20x20, 12x12, 4x4, 2x2, 1x1
After : 580x580, 292x292, 148x148, 76x76, 40x40, 20x20, 12x12, 8x8, 4x4, 2x2, 1x1

In the example above, descending layers have [better] dimensions and an additional level is created which fills the rather large gap between 12x12 and 4x4. Because of the nature of DDS, it is impossible to always have mipmaps that are a perfect halve of the previous level. The new method should create a better transition between mipmap dimensions for DDS textures in nearly all cases.
